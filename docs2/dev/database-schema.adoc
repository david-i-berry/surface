= Database Schema
:imagesdir: ../images

This chapter documents the SURFACE database structure.

== Database Overview

SURFACE uses PostgreSQL with two key extensions:

* *TimescaleDB* - Time-series data optimization
* *PostGIS* - Geographic/spatial queries

== Core Entity Relationships

----
Country
   └── AdministrativeRegion
          └── District
                 └── Station
                        ├── StationVariable → Variable → Unit
                        ├── Equipment → EquipmentType → Manufacturer
                        ├── MaintenanceReport → Technician
                        ├── RawData (hypertable)
                        ├── HourlySummary (hypertable)
                        ├── DailySummary (hypertable)
                        ├── MonthlySummary
                        └── YearlySummary
----

== TimescaleDB Configuration

=== Hypertable Setup

[source,sql]
----
SELECT create_hypertable('wx_rawdata', 'datetime');
SELECT create_hypertable('wx_hourlysummary', 'datetime');
SELECT create_hypertable('wx_dailysummary', 'day');
----

=== Compression Policy

[source,sql]
----
SELECT add_compression_policy('wx_rawdata', INTERVAL '7 days');
----

=== Retention Policy

[source,sql]
----
SELECT add_retention_policy('wx_rawdata', INTERVAL '2 years');
----

== Index Recommendations

Key indexes for query performance:

[source,sql]
----
-- Station-time queries
CREATE INDEX idx_rawdata_station_datetime
    ON wx_rawdata (station_id, datetime DESC);

-- Variable queries
CREATE INDEX idx_rawdata_variable_datetime
    ON wx_rawdata (variable_id, datetime DESC);

-- Geographic queries
CREATE INDEX idx_station_location
    ON wx_station USING GIST (location);
----
