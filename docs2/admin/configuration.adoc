= Configuration
:imagesdir: ../images

This chapter covers SURFACE configuration options and settings management.

== Configuration Overview

SURFACE is configured through:

* *Environment variables* - Core system settings in `production.env`
* *Frontend Settings menu* - User-accessible configuration
* *Django Admin* - Advanced administrative settings

== Environment Variables

The `production.env` file contains critical system configuration. This section documents all available variables.

=== Complete Variable Reference

==== PostgreSQL Database

[cols="2,1,3"]
|===
|Variable |Required |Description

|`POSTGRES_PASSWORD`
|Yes
|PostgreSQL superuser password

|`POSTGRES_USER`
|Yes
|PostgreSQL username (default: postgres)

|`POSTGRES_DB`
|Yes
|Database name to create

|`PGDATA`
|No
|PostgreSQL data directory (default: `/var/lib/postgresql/data/pgdata`)

|`TIMESCALEDB_TELEMETRY`
|No
|Disable TimescaleDB telemetry (set to `off`)
|===

==== Django Database Connection

[cols="2,1,3"]
|===
|Variable |Required |Description

|`SURFACE_DB_ENGINE`
|Yes
|Database engine (use `django.contrib.gis.db.backends.postgis`)

|`SURFACE_DB_HOST`
|Yes
|Database hostname (use `postgres` for Docker)

|`SURFACE_DB_PORT`
|Yes

|`SURFACE_DB_NAME`
|Yes

|`SURFACE_DB_USER`
|Yes
|Database username (must match `POSTGRES_USER`)

|`SURFACE_DB_PASSWORD`
|Yes
|Database password (must match `POSTGRES_PASSWORD`)
|===

==== Security and Secrets

[cols="2,1,3"]
|===
|Variable |Required |Description

|`SURFACE_SECRET_KEY`
|Yes
|Django secret key for cryptographic signing. Generate with: `python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"`

|`SURFACE_SECRET_ENCRYPTION_KEY`
|Yes
|44-character Base64 Fernet key for encrypting sensitive data. Generate with: `python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"`

|`SURFACE_DJANGO_DEBUG`
|No
|Enable Django debug mode (`True`/`False`). Always `False` in production.
|===

IMPORTANT: Generate unique values for production. Never use example or default keys. The encryption key must remain constant; changing it will make existing encrypted data unreadable.

=== Example production.env

[source,properties]
----
# Database
TIMESCALEDB_TELEMETRY=off
POSTGRES_PASSWORD=SecurePassword123!
POSTGRES_USER=surface_user
POSTGRES_DB=surface_db
PGDATA=/var/lib/postgresql/data/pgdata

# Django Database Connection
SURFACE_DB_ENGINE=django.contrib.gis.db.backends.postgis
SURFACE_DB_HOST=postgres
SURFACE_DB_PORT=5432
SURFACE_DB_NAME=surface_db
SURFACE_DB_USER=surface_user
SURFACE_DB_PASSWORD=SecurePassword123!

# Security (generate unique values!)
SURFACE_SECRET_KEY=your-50-character-random-string-here
SURFACE_SECRET_ENCRYPTION_KEY=your-44-character-base64-fernet-key

# Application
SURFACE_DATA_DIR=/data
SURFACE_BROKER_URL=redis://redis:6379/0
SURFACE_DJANGO_DEBUG=False
NUMBER_OF_GUNICORN_WORKERS=2

# Timezone
TIMEZONE_NAME=America/Belize
TIMEZONE_OFFSET=-360

# Map Center (Belize)
MAP_LATITUDE=17.1899
MAP_LONGITUDE=-88.4976
MAP_ZOOM=8

# Spatial Analysis Bounds (Belize)
SPATIAL_ANALYSIS_INITIAL_LATITUDE=18.5

SPATIAL_ANALYSIS_INITIAL_LONGITUDE=-89.3

# Variable IDs (adjust for your installation)
STATION_MAP_WIND_SPEED_ID=20
STATION_MAP_WIND_GUST_ID=21
STATION_MAP_WIND_DIRECTION_ID=22
STATION_MAP_TEMP_MAX_ID=13
STATION_MAP_TEMP_MIN_ID=14

== Frontend Configuration

Upload geographic data files:

1. Click the upload area for each file type
2. Select or drag-drop the appropriate file
3. Click *Save* to apply changes
4. Refresh the map to see updates

== Scheduled Tasks

SURFACE uses Celery Beat for scheduled operations.

=== Viewing Scheduled Tasks

1. Go to *Administrative Area*
2. Navigate to *Periodic Tasks*
3. Review configured schedules

=== Common Scheduled Tasks

[cols="1,2,2"]
|===
|Task |Purpose |Typical Schedule

|Data aggregation
|Calculate hourly/daily summaries
|Every 30 minutes

|FTP ingestion
|Pull data from FTP servers
|Every 5-15 minutes

|WIS2 publishing
|Send data to WIS2Box
|Hourly (minute offset)

|Database backup
|Create backup files
|Daily/Twice daily

|Cache cleanup
|Clear old cached data
|Daily
|===

=== Creating Custom Schedules

1. Create a schedule in *Crontabs* or *Intervals*
2. Create a *Periodic Task* referencing the schedule
3. Configure task parameters
4. Enable the task

== Backup Configuration

Configure automated backups in Django Admin.

=== Creating Backup Tasks

1. Navigate to *Backup Tasks*
2. Click *Add Backup Task*
3. Configure:
+
[cols="1,3"]
|===
|Field |Description

|Name
|Descriptive name for the backup

|Cron Schedule (UTC)
|When to run (cron format)

|Backup file name
|Name pattern for backup files

|Retention duration (days)
|How long to keep backups

|FTP server
|Where to store backups

|Remote folder
|FTP directory path

|Active
|Enable/disable the task
|===

4. Click *Save*

== Performance Tuning

=== Database

* Increase PostgreSQL shared_buffers for larger datasets
* Configure TimescaleDB chunk sizes for optimal performance
* Regular VACUUM and ANALYZE maintenance

=== Caching

* Memcached is configured by default
* Increase cache memory for heavily used systems
* Configure cache timeout based on data update frequency

=== Celery Workers

* Increase worker count for high-volume data processing
* Configure task queues for priority processing
* Monitor task queue lengths

== Configuration Best Practices

1. *Document changes* - Keep a log of configuration changes
2. *Test in staging* - Test configuration changes before production
3. *Backup before changes* - Create backup before major configuration changes
4. *Use version control* - Track environment file changes (without secrets)
5. *Regular review* - Periodically review and optimize configuration
