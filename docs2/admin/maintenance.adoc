= Equipment and Maintenance
:imagesdir: ../images

This chapter covers equipment inventory management and maintenance reporting.

== Maintenance Overview

SURFACE provides tools for:

* Tracking equipment inventory
* Recording maintenance visits
* Documenting station condition
* Planning future maintenance

== Equipment Inventory

Navigate to *Maintenance* > *Equipment Inventory*.

=== Viewing Inventory

The inventory list shows all registered equipment with:

* Equipment type
* Model
* Serial number
* Status
* Current location (station)

=== Adding New Equipment

1. Click *New Equipment* button
2. Fill out the equipment form
3. Click *Save*

==== Equipment Form Fields

[cols="1,3"]
|===
|Field |Description

|Equipment Type
|Category (sensor, logger, power supply, etc.)

|Manufacturer
|Equipment manufacturer

|Model
|Model name/number

|Serial Number
|Unique identifier

|Acquisition Date
|When equipment was received

|Status
|Current condition

|Notes
|Additional information
|===

=== Equipment Best Practices

TIP: Enter new equipment into SURFACE immediately upon receipt. This begins the audit trail before field deployment.

Recommended workflow:

1. Receive equipment
2. Enter into SURFACE inventory
3. Test and calibrate
4. Document calibration in equipment record
5. Deploy to station
6. Update location in inventory
7. Create maintenance report for installation

== Maintenance Reports

Navigate to *Maintenance* > *Maintenance Reports*.

=== Report List

View existing reports with:

* Station name
* Visit date
* Next scheduled visit
* Station profile and type
* Technician name
* Visit type
* Status (Draft, Published, Approved)
* Actions (View, Edit, Delete)

=== Filtering Reports

* Date range selector
* Station type filter (Manual/Automatic)
* Search by station name

=== Report Status Workflow

[cols="1,3"]
|===
|Status |Description

|Draft
|Report in progress, can be edited

|Published
|Submitted for approval

|Approved
|Finalized, no further edits
|===

== Creating a Maintenance Report

=== Prerequisites

Before creating reports, configure:

* *Report Templates* - Define which equipment types to include
* *Technicians* - Add technician names in Django Admin
* *Equipment* - Ensure relevant equipment is in inventory

=== Step 1: Initial Information

Click *Create New Report* and fill in:

[cols="1,3"]
|===
|Field |Description

|Type of Visit
|Installation, Corrective, Inspection, Preventative, Replacement

|Station Name
|Select from dropdown

|Responsible Technician
|Primary technician conducting visit

|Date of Visit
|When visit occurred

|Initial Time
|Arrival time at station
|===

Click *Create Report* when all fields are complete.

=== Step 2: Initial Station Description

Describe the station condition upon arrival:

* Note any issues observed
* Document general station state
* Upload photos showing condition

Click *Save and Continue*.

=== Step 3: Equipment Inspection

For each equipment type in the template:

1. Select the specific equipment (by serial number)
2. Rate working condition:
** Fully Functional
** Partially Functional
** Not Functional
3. Describe the equipment state
4. Upload photos
5. Click *Save and Continue*

Repeat for all equipment types.

NOTE: First-time selection of equipment in a report begins its audit trail from receipt to field installation.

=== Step 4: Station Contacts

Document host/location contact information:

* Contact name
* Phone number
* Email
* Address
* Special access instructions

=== Step 5: Summary of Visit

Complete the report with:

[cols="1,3"]
|===
|Field |Description

|Other Technicians
|Additional team members present

|Work Summary
|Description of work performed

|Future Work
|Tasks needed on next visit

|Next Visit Date
|Scheduled return date

|End Time
|When visit concluded

|Data Logger Program
|Associated logger configuration file
|===

=== Step 6: Finalize Report

Choose an action:

[cols="1,3"]
|===
|Option |Description

|Save Draft
|Save for later editing

|Preview
|View as PDF

|Publish
|Submit for supervisor approval
|===

== Approving Reports

Supervisors with appropriate permissions can approve published reports.

=== Approval Process

1. View maintenance reports list
2. Identify reports with "Published" status
3. Click the approval checkmark icon
4. Review the report content
5. Confirm approval in dialog

Once approved, reports cannot be edited.

== Report Templates

Configure what equipment types appear in maintenance reports.

=== Managing Templates

1. Access Django Admin
2. Navigate to *Station profile equipment types*
3. Associate equipment types with station profiles

This ensures:
* Meteorological stations show met sensors
* Hydrological stations show hydro equipment
* Custom profiles show relevant equipment

== Technician Management

Add technicians in Django Admin:

1. Navigate to *Technicians*
2. Click *Add Technician*
3. Enter name and contact information
4. Save

Technicians appear in report dropdowns after creation.

== Equipment Audit Trail

Track equipment lifecycle:

1. *Receipt* - Entered into inventory
2. *Calibration* - Documented in equipment notes
3. *Deployment* - First maintenance report at station
4. *Service* - Subsequent maintenance reports
5. *Replacement* - Report documents swap
6. *Retirement* - Status changed, reason documented

== Maintenance Planning

=== Scheduling

Use *Next Visit Date* field to plan maintenance:

* Set realistic intervals
* Consider travel logistics
* Account for seasonal factors

=== Viewing Schedule

Filter reports by:
* Upcoming next visit dates
* Stations due for maintenance
* Overdue visits

=== Best Practices

1. *Regular schedules* - Establish routine visit intervals
2. *Complete documentation* - Fill all report sections
3. *Photo evidence* - Document condition with images
4. *Prompt reporting* - Create reports immediately after visits
5. *Supervisor review* - Always get reports approved
6. *Equipment tracking* - Keep inventory current
