== Introduction

*SURFACE* - **S**ystem for **U**nified **R**eal-time monitoring and **F**orecasting of **A**tmospheric and **C**limatic **E**vents

SURFACE is a comprehensive weather and hydrological data management system developed by the National Meteorological Service of Belize (NMS). It provides tools for collecting, processing, storing, and analyzing environmental data from distributed monitoring stations.

=== Key Features

* Real-time data ingestion via FTP and NOAA LRGS
* Multiple data entry methods (key entry, Excel, AWS files)
* Aggregation of sub-hourly data to hourly, daily, monthly, and yearly summaries
* Monitoring of weather station performance and data flow
* Metadata storage for stations, instruments, and maintenance reports
* Automatic Quality Control (AQC) procedures - Range, Step, and Persistence checks
* WMO message generation - GBON/SYNOP
* Automatic data push to WIS2Box/WIS2.0 node
* WMO OSCAR integration for metadata management

=== Architecture Overview

SURFACE consists of seven main Docker containers working together:

[cols="1,3"]
|===
|Component |Description

|Nginx
|Web server and reverse proxy

|Django/Python
|Web API and codebase (RESTful)

|PostgreSQL/TimescaleDB
|Time-series database with PostGIS

|Memcached
|Caching layer for database results

|Celery Worker
|Background data processing

|Celery Beat
|Task scheduler

|Redis
|Message broker
|===
