= Data Management
:imagesdir: ../images

This chapter covers data capture, import, and export functions in SURFACE.

== Data Menu Overview

The Data menu provides access to:

* *Monthly Capture* - Manual daily data entry
* *Synop Capture* - Synoptic observation entry
* *Data Inventory* - View data availability
* *Import* - Upload data files
* *Export* - Download data

== Monthly Capture

The Monthly Capture page allows manual entry of daily observation data.

=== Entering Data

1. Navigate to *Data* > *Monthly Capture*
2. Select the station from the dropdown menu
3. Choose the date using the calendar picker
4. Click *LOAD* to retrieve existing data or open a blank form
5. Enter or modify values in the form fields
6. Click *SAVE* to store the data

=== Form Controls

[cols="1,3"]
|===
|Button |Function

|LOAD
|Retrieve data for selected station and date

|CANCEL
|Clear selections and reset the form

|SAVE
|Store entered data (active only when valid data exists)
|===

== Synop Capture

For stations performing manual synoptic observations.

=== Entering Synoptic Data

1. Navigate to *Data* > *Synop Capture*
2. Select the synoptic station (required)
3. Choose the observation date
4. Click *LOAD* to open the data entry form
5. Enter synoptic observation values
6. Save the data

NOTE: Only stations configured as "Synoptic Stations" appear in the station dropdown.

== Data Import

Import data from Excel files (XLSX format) into SURFACE.

=== Import Process

1. Navigate to *Data* > *Import*
2. Click "Choose file" or drag and drop files into the attachment box
3. Configure import options
4. Click *Upload*

=== Import Options

[cols="1,3"]
|===
|Option |Description

|Override Data
|When enabled, new data replaces existing records for matching dates

|Reset
|Clears file selections to start over
|===

=== File Requirements

* Format: `.xlsx` (Excel) only
* Maximum file size: 2 MB per file
* Maximum upload: 4 files (8 MB total)

=== Viewing Import History

The *Manual Data Files Table* displays:

* File Name
* Upload Date
* Processing Status
* Actions (delete option)

Use pagination controls to navigate through multiple records.

== Data Export

Export station data for analysis or sharing.

=== Export Process

1. Navigate to *Data* > *Export*
2. Apply filters to select data
3. Configure export settings
4. Click *Export* or *Combine and Download .XLSX*

=== Station Filters

[cols="1,3"]
|===
|Filter |Purpose

|District
|Filter by administrative district

|Watershed
|Filter by watershed region

|Profile
|Filter by station profile type

|Station
|Select specific station(s)

|Active
|Toggle to show only active stations

|Automatic
|Toggle to show only automatic stations
|===

=== Data Settings

[cols="1,3"]
|===
|Setting |Options

|Data Source
|Raw data, Hourly summary, Daily summary, etc.

|Data Interval
|5 minutes, 15 minutes, hourly, daily, etc.

|Variables
|Select which measurements to include
|===

=== Date and Time Settings

* *Initial Date/Time* - Start of export period
* *Final Date/Time* - End of export period
* *UTC Toggle* - Export in Coordinated Universal Time or local time

=== Export Options

[cols="1,3"]
|===
|Option |Description

|Export
|Generate individual export file

|Combine and Download .XLSX
|Merge all selected data into single Excel file
|===

=== Export Status

The data table shows export requests with:

* *Id* - Unique identifier
* *Station* - Associated station
* *Prepared by* - User who requested export
* *Requested at* - Request timestamp
* *Ready at* - Completion timestamp
* *Status* - Processing status (Ready, Processing, etc.)
* *Lines* - Number of data rows
* *Actions* - Download button when ready

== Best Practices

=== For Data Entry
* Verify station selection before entering data
* Double-check values before saving
* Use consistent units as configured in the system
* Note any data quality issues in remarks fields

=== For Import
* Ensure Excel files follow the expected format
* Back up existing data before using override option
* Review import status for any errors

=== For Export
* Start with smaller date ranges to verify data
* Use UTC for international data sharing
* Include quality flags when data validation is important
